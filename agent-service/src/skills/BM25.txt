【核心规则】
1. 提取核心词（Extraction）：精准提取原问题中的核心名词、动词、专有名词、特定业务概念。
2. 同义词及术语扩展（Expansion）：自然语言往往偏口语，请务必根据“公司政策、规章制度”的语境，补充官方书面语、法律术语及常见同义词。
   - 例：“辞退” -> “解除劳动合同 离职 辞职”
   - 例：“发钱” -> “发放 补偿金 结算 薪酬津贴”
3. 标题预测（Title Guessing）：预判包含该答案的政策文档标题可能包含的词汇，补充进词袋中。
   - 例：“管理办法”、“制度”、“规范”、“标准”、“员工手册”、“指引”、“违规处理”。
4. 严格去噪（Denoising）：**绝对不能**包含自然语言停用词、疑问词、代词和语气词（如：的、了、吗、能不能、怎么、如果、我、去、哎呀）。
5. 格式要求：所有提取和扩展的词汇，用**单个空格**隔开，组成一个长字符串。

文档的文件名列表如下，供核心词猜测参考：
办公行为规范、设备管理办法、差旅报销制度、休假管理规定、信息安全规范、薪酬福利制度、绩效考核制度、招聘与入职流程。

示例1：
用户输入："我是研发部的，去年年假没休完，今年底能折现发工资里吗？"
改写为："休假 年休假 年假 带薪假 未休 没休完 折现 补偿金 结算 工资 薪酬福利 考勤制度 员工手册 管理办法 休假管理"

示例2：
用户输入："出差去上海的高铁票可以报销一等座吗？"
改写为："差旅 出差 交通 报销 报销流程 额度 标准 差旅报销 费用 费用报销"


【核心改写策略】
向量检索依赖语义空间的距离。请不要生成仅仅是"同义词替换"的重复问题，必须对同一个原问题按以下两个层面生成多个差异化的query：

1. Query 0：意图提纯
   - 目标：用标准、书面的官方语态，精准重述用户的核心意图
   - 强化：核心意图关键词（差旅标准、住宿报销、病假待遇、违规处理等），也可以扩展同义词及其术语

2. Query 1~N：按文档改写
   - 目标：针对原问题涉及的每个文档，用该文档的术语和视角分别改写
   - 规则：每个query只针对一个文档，扩展相关的同义词，尽可能不跨文档
   - 示例：
     * 针对差旅报销制度 → 用"差旅、差旅费、报销、住宿标准、交通"等术语
     * 针对休假管理规定 → 用"休假、假期、病假待遇、年假天数、请假审批"等术语
   - 数量：根据问题涉及的文档数量决定（通常2-4个）

【文档速查】

01_办公行为规范
  覆盖：考勤管理、办公环境、会议纪律、着装规范、沟通规范、本文档规范违规处理等

02_设备管理办法
  覆盖：设备领用、设备使用、资源借用、设备报废、本文档规范违规处理等

03_差旅报销制度
  覆盖：差旅申请、交通报销、住宿报销、报销流程、本文档规范违规处理等

04_休假管理规定
  覆盖：带薪年假、病假事假、其他假期、休假审批、本文档规范违规处理等

05_信息安全规范
  覆盖：密码管理、数据保护、网络安全、邮件安全、本文档规范违规处理等

06_薪酬福利制度
  覆盖：薪酬结构、福利待遇、薪酬管理、薪酬保密等

07_绩效考核制度
  覆盖：考核原则、考核周期、考核内容、考核等级、结果应用、申诉等

08_招聘与入职流程
  覆盖：招聘计划、招聘流程、入职流程、特殊招聘等

【示例】（高复杂度+多子问题，涉及3个文档）

用户输入："公司派我去成都出差半个月，需要乘坐高铁，请问我可以坐一等座吗？住宿标准是多少？另外出差期间的餐补和交通补是怎么计算的？如果借用公司的笔记本电脑在酒店办公，设备出现故障了应该联系谁报修？"

Query 0（意图提纯）：
"员工长期出差交通等级标准住宿报销标准及借用设备故障报修"

Query 1（针对差旅报销制度）：
"长期出差员工高铁座位等级限制住宿补贴餐补交通补标准"

Query 2（针对薪酬福利制度）：
"员工差旅津贴餐补交通补计算方式及发放标准"

Query 3（针对设备管理办法）：
"出差借用笔记本电脑故障报修流程及IT部门响应时间"
