# 信息安全规范

## 1. 总则

### 1.1 目的
为保障公司信息安全，保护公司商业秘密和客户信息，规范员工信息安全行为，防范信息安全风险，根据《中华人民共和国网络安全法》、《中华人民共和国数据安全法》、《中华人民共和国个人信息保护法》等法律法规，结合公司实际情况，特制定本规范。本规范适用于公司全体员工及所有访问公司信息系统的人员。

### 1.2 适用范围
本规范适用于公司全体员工，包括正式员工、试用期员工、实习生、外包人员及其他与公司有业务往来的人员。本规范所指的信息包括但不限于：
- 公司商业秘密、技术秘密、经营信息
- 客户信息、合作伙伴信息
- 员工个人信息
- 公司财务信息、人事信息
- 公司信息系统、网络设备、服务器等
- 其他公司认为需要保密的信息

### 1.3 管理职责
- **信息安全委员会**：负责公司信息安全战略规划、重大决策和资源保障
- **IT部门**：负责信息安全技术保障、系统安全运维、安全事件处置
- **人力资源部**：负责信息安全教育培训、员工信息安全考核
- **各部门负责人**：负责本部门信息安全管理，监督本规范在本部门的执行
- **全体员工**：负责遵守本规范，保护公司信息安全，及时报告信息安全事件

## 2. 密码管理

### 2.1 密码设置
- **密码长度**：
  - 系统登录密码长度至少8位
  - 重要系统（财务系统、人事系统等）密码长度至少12位
  - 服务器、网络设备等核心设备密码长度至少16位

- **密码复杂度**：
  - 必须包含大小写字母
  - 必须包含数字
  - 必须包含特殊字符（如!@#$%^&*等）
  - 不得使用连续字符（如123456、abcdef等）
  - 不得使用重复字符（如aaaaaa、111111等）

- **密码示例**：
  - 弱密码：password、123456、admin、qwerty
  - 强密码：Kj9#mP2$xL5@、Qw3$Er5%Ty7^、Zx8&Cv4*Bn2(

### 2.2 密码使用
- **定期更换**：
  - 普通系统密码每90天更换一次
  - 重要系统密码每60天更换一次
  - 核心设备密码每30天更换一次
  - 密码更换后应妥善保管，不得重复使用最近5次的密码

- **密码保管**：
  - 密码应妥善保管，不得告诉他人
  - 不得将密码写在纸上或贴在电脑上
  - 不得将密码保存在浏览器或文档中
  - 建议使用密码管理软件（如1Password、LastPass等）保管密码

- **密码分享**：
  - 严禁与他人分享个人密码
  - 确因工作需要共享密码的，应经部门主管和IT部门批准
  - 共享密码应设置使用期限，到期及时更换
  - 密码共享应做好记录，注明共享对象、用途、期限等

### 2.3 密码安全
- **密码泄露处理**：
  - 发现密码可能泄露时，应立即更换密码
  - 及时向IT部门报告，说明泄露原因和可能的影响
  - 配合IT部门进行安全检查和风险评估
  - 如造成损失，应承担相应责任

- **多因素认证**：
  - 重要系统应启用多因素认证
  - 多因素认证包括：短信验证码、邮件验证码、动态令牌、生物识别等
  - 多因素认证设备应妥善保管，不得转借他人
  - 发现多因素认证设备丢失应立即报告IT部门

## 3. 数据保护

### 3.1 数据分类
- **绝密数据**：
  - 定义：涉及公司核心商业秘密，泄露会给公司造成重大损失的数据
  - 范围：公司核心技术、重大经营决策、关键客户信息、未公开的财务数据等
  - 访问权限：仅限核心管理层和经特别授权的人员
  - 保护要求：最高级别保护，加密存储，严格访问控制，全程审计

- **机密数据**：
  - 定义：涉及公司重要商业秘密，泄露会给公司造成较大损失的数据
  - 范围：公司技术资料、客户名单、供应商信息、财务报表、人事信息等
  - 访问权限：仅限部门内部授权人员
  - 保护要求：高级别保护，加密存储，访问控制，定期审计

- **内部数据**：
  - 定义：公司内部使用，不对外公开的数据
  - 范围：内部管理制度、工作流程、日常办公文档等
  - 访问权限：公司全体员工
  - 保护要求：中等级别保护，访问控制，定期备份

- **公开数据**：
  - 定义：可对外发布的数据
  - 范围：公司宣传资料、公开招聘信息、已发布的新闻等
  - 访问权限：可对外公开
  - 保护要求：低级别保护，确保数据准确性

### 3.2 数据存储
- **加密存储**：
  - 绝密数据和机密数据必须加密存储
  - 加密算法应使用国家认可的加密算法
  - 加密密钥应妥善保管，定期更换
  - 加密设备应定期检查，确保正常运行

- **存储介质**：
  - 重要数据应存储在公司指定的服务器或存储设备上
  - 不得将重要数据存储在个人电脑、U盘、移动硬盘等个人设备上
  - 存储介质应定期检查，确保数据安全
  - 报废的存储介质应进行数据销毁，防止数据泄露

- **数据备份**：
  - 重要数据应定期备份，备份频率根据数据重要性确定
  - 绝密数据每天备份一次
  - 机密数据每周备份一次
  - 内部数据每月备份一次
  - 备份数据应异地存储，定期检查备份数据的可用性

### 3.3 数据传输
- **传输加密**：
  - 绝密数据和机密数据传输必须加密
  - 应使用HTTPS、SFTP、VPN等安全传输协议
  - 不得使用明文传输重要数据
  - 传输过程中应确保数据完整性，防止数据被篡改

- **传输渠道**：
  - 应使用公司指定的传输渠道传输数据
  - 不得通过私人邮箱、微信、QQ等传输重要数据
  - 确需通过外部渠道传输的，应进行加密处理
  - 大型数据传输应使用公司指定的文件传输工具

- **传输审批**：
  - 绝密数据对外传输需总经理审批
  - 机密数据对外传输需分管副总审批
  - 内部数据跨部门传输需部门主管审批
  - 数据传输应做好记录，注明传输对象、内容、时间等

## 4. 网络安全

### 4.1 网络接入
- **公司网络接入**：
  - 只能连接公司授权的网络
  - 连接公司网络前应确保设备安全
  - 应使用公司分配的账号密码登录
  - 不得使用他人账号登录公司网络

- **无线网络接入**：
  - 只能连接公司授权的WiFi网络
  - 公司WiFi网络应使用WPA3或WPA2-PSK加密
  - 不得连接未授权的WiFi网络
  - 不得使用手机热点连接公司设备

- **远程办公接入**：
  - 远程办公应使用公司指定的VPN
  - VPN连接应使用多因素认证
  - 远程办公结束后应及时断开VPN连接
  - 不得在公共WiFi环境下进行远程办公

### 4.2 设备安全
- **电脑安全**：
  - 公司电脑必须设置开机密码
  - 离开工位时应锁定电脑屏幕
  - 电脑闲置超过10分钟应自动锁定
  - 下班时应关闭电脑及相关设备电源

- **软件安装**：
  - 只能安装公司授权的软件
  - 软件安装应向IT部门申请，由IT部门统一安装
  - 不得安装盗版软件、破解软件、游戏软件等
  - 不得安装来源不明的软件

- **系统更新**：
  - 应及时安装系统安全更新和补丁
  - IT部门会定期推送系统更新，应及时安装
  - 更新前应备份重要数据，避免数据丢失
  - 如更新后出现问题，应及时联系IT部门

### 4.3 网络行为
- **禁止行为**：
  - 禁止访问非法网站、色情网站、赌博网站等
  - 禁止下载来源不明的文件
  - 禁止使用公司网络进行挖矿、刷单、炒币等活动
  - 禁止在公司网络上发布违法违规信息

- **安全检测**：
  - IT部门会定期进行网络安全检测
  - 发现安全隐患会及时通知相关人员整改
  - 员工应配合IT部门进行安全检查
  - 不得拒绝或阻挠安全检测工作

- **异常处理**：
  - 发现设备异常（如运行缓慢、弹出异常窗口等）应及时联系IT部门
  - 发现网络异常应及时报告IT部门
  - 疑似感染病毒应立即断开网络连接
  - 配合IT部门进行病毒查杀和系统恢复

## 5. 邮件安全

### 5.1 邮箱使用
- **邮箱账号**：
  - 只能使用公司分配的邮箱进行工作沟通
  - 不得使用私人邮箱处理工作事务
  - 邮箱密码应按密码管理规范设置和更换
  - 不得将邮箱账号密码告诉他人

- **邮箱登录**：
  - 只能在公司授权的设备上登录公司邮箱
  - 登录邮箱应使用安全的网络环境
  - 登录后应及时退出，特别是在公共设备上
  - 发现邮箱异常登录应及时报告IT部门

- **邮箱清理**：
  - 应定期清理邮箱，删除不需要的邮件
  - 重要邮件应及时归档保存
  - 定期检查垃圾邮件文件夹，避免误删重要邮件
  - 离职时应移交邮箱权限，删除个人邮件

### 5.2 邮件内容
- **邮件内容规范**：
  - 邮件内容应简洁明了，主题明确
  - 邮件应使用规范的用语，尊重他人
  - 邮件应注明收件人、抄送人、密送人
  - 重要邮件应要求收件人确认收到

- **敏感信息处理**：
  - 不得在邮件中发送绝密数据和机密数据
  - 确需发送的应进行加密处理
  - 敏感信息应使用公司指定的加密传输方式
  - 发送敏感信息应经部门主管审批

- **附件管理**：
  - 邮件附件应病毒查杀后再发送
  - 附件大小应控制在合理范围内
  - 大型附件应使用公司指定的文件传输工具
  - 发送附件前应确认收件人正确

### 5.3 钓鱼邮件防范
- **识别钓鱼邮件**：
  - 注意查看发件人邮箱地址，警惕陌生发件人
  - 注意邮件内容，避免诱惑性、威胁性语言
  - 注意邮件链接，不要点击可疑链接
  - 注意邮件附件，不要打开可疑附件

- **处理可疑邮件**：
  - 收到可疑邮件应及时报告IT部门
  - 不要点击邮件中的链接
  - 不要下载邮件中的附件
  - 不要回复可疑邮件

- **防范措施**：
  - 公司会定期进行钓鱼邮件演练
  - 员工应积极参加钓鱼邮件培训
  - 提高安全意识，增强识别能力
  - 发现钓鱼邮件及时举报

## 6. 信息安全事件处置

### 6.1 事件报告
- **报告渠道**：
  - 发现信息安全事件应立即向IT部门报告
  - 可通过电话、邮件、即时通讯等方式报告
  - 紧急情况可直接向信息安全委员会报告
  - 报告应真实、准确、完整

- **报告内容**：
  - 事件发生的时间、地点
  - 事件的详细情况
  - 可能造成的影响
  - 已采取的措施
  - 需要的支持

- **报告时限**：
  - 一般事件应在1小时内报告
  - 重要事件应在30分钟内报告
  - 紧急事件应立即报告
  - 不得迟报、谎报、瞒报

### 6.2 事件处置
- **应急响应**：
  - IT部门接到报告后应立即启动应急响应
  - 根据事件等级采取相应的处置措施
  - 控制事态发展，防止事件扩大
  - 保护现场，保留证据

- **处置措施**：
  - 切断攻击源，阻止攻击继续
  - 修复系统漏洞，恢复系统正常运行
  - 评估损失，制定恢复方案
  - 通知相关人员，做好沟通协调

- **事件调查**：
  - 成立事件调查组，开展调查工作
  - 查明事件原因，确定事件责任
  - 评估事件影响，总结经验教训
  - 形成调查报告，提出改进建议

### 6.3 事件追责
- **责任认定**：
  - 根据事件调查结果，认定事件责任
  - 区分故意行为和过失行为
  - 区分主要责任和次要责任
  - 形成责任认定报告

- **追责处理**：
  - 对责任人进行批评教育
  - 情节较重的给予纪律处分
  - 造成损失的要求赔偿
  - 涉嫌犯罪的移送司法机关

- **改进措施**：
  - 根据事件教训，完善管理制度
  - 加强技术防护，提高安全能力
  - 加强教育培训，提高安全意识
  - 定期检查评估，持续改进提升

## 7. 附则

### 7.1 规范执行
- 本规范自发布之日起施行，原《信息安全规范》同时废止
- IT部门负责本规范的解释和修订
- 每年对本规范的执行情况进行评估，根据需要进行修订

### 7.2 考核与奖惩
- 信息安全纳入员工绩效考核
- 对信息安全工作表现突出的给予表彰奖励
- 对违反本规范的给予批评教育、纪律处分
- 造成损失的要求赔偿，涉嫌犯罪的移送司法机关

### 7.3 其他
- 员工应认真学习本规范，严格遵守各项规定
- 各部门应定期组织信息安全培训和演练
- IT部门应定期进行信息安全检查和评估
- 公司将持续投入资源，提升信息安全保障能力
