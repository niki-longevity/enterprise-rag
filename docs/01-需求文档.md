# 企业内部员工助手智能体 - 需求文档

## 1. 项目概述

### 1.1 项目背景
为企业员工提供一站式智能助手，通过自然语言对话解决政策咨询、资源查询、审批申请等日常办公需求。

### 1.2 项目目标
- 实现三条核心链路：政策问答、资源查询、审批申请
- 提供友好的对话式交互界面
- 展示LangGraph智能编排与RAG技术的结合应用
- 作为简历项目，体现完整的全栈开发能力

---

## 2. 用户角色

| 角色 | 描述 |
|------|------|
| 普通员工 | 使用助手查询政策、申请资源 |
| 负责人（模拟） | 审批工单，项目中用模拟逻辑实现 |

---

## 3. 功能需求

### 3.1 核心链路一：IT政策问答（RAG）

**功能描述**：员工通过自然语言询问公司政策，系统基于RAG检索相关文档并回答。

**典型场景**：
- "我能不能带宠物上班？"
- "我想换个新电脑，公司的相关规定是什么？"
- "请假流程是怎样的？"

**验收标准**：
- [ ] 支持至少5个政策文档的检索
- [ ] 回答需引用对应的政策来源
- [ ] 当问题超出政策范围时，应礼貌告知
- [ ] 响应时间 < 5秒

---

### 3.2 核心链路二：库存与资源查询

**功能描述**：员工查询公司可借用的物资和资源库存。

**典型场景**：
- "我下周要办活动，公司现在还有没有闲置的投影仪可以借？"
- "我想申请使用IDEA旗舰版，还有多余的激活码吗？"
- "会议室A明天下午3点有空吗？"

**验收标准**：
- [ ] 支持查询投影仪、笔记本电脑、会议室、软件激活码等资源
- [ ] 能准确返回资源的可用/不可用状态
- [ ] 支持按时间查询会议室可用性
- [ ] 响应时间 < 3秒

---

### 3.3 核心链路三：审批流

**功能描述**：员工通过对话申请借用资源或申请权限，系统自动生成工单并模拟审批流程。

**典型场景**：
- "帮我申请借用一台投影仪，下周二用"
- "帮我开一下业务数据库的只读权限，我查个数据"
- "我需要借会议室A，这周五下午2点到4点"

**验收标准**：
- [ ] 支持多任务拆解（一个请求包含多个审批项）
- [ ] 自动生成工单号和工单详情
- [ ] 模拟负责人审批（70%通过率，30%拒绝）
- [ ] 员工可查询工单状态
- [ ] 工单创建时间 < 5秒

## 4. 页面设计

### 4.1 页面清单

| 页面名称 | 路由 | 说明 |
|---------|------|------|
| 聊天主页面 | `/` | 核心对话界面 |

### 4.2 聊天主页面（核心页面）

**布局结构**：
```
┌─────────────────────────────────────────────────┐
│  企业员工助手  🤖                          │
├─────────────────────────────────────────────────┤
│                                                 │
│  [历史消息区域]                                 │
│  ┌─────────────────────────────────────────┐   │
│  │ 员工：我能不能带宠物上班？              │   │
│  └─────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────┐   │
│  │ 助手：根据《员工办公规范》第3.2条...    │   │
│  │ 📄 来源：员工办公规范.md                │   │
│  └─────────────────────────────────────────┘   │
│                                                 │
│  [输入区域]                                     │
│  ┌─────────────────────────────────────────┐   │
│  │ 输入您的问题...                   [发送] │   │
│  └─────────────────────────────────────────┘   │
│                                                 │
└─────────────────────────────────────────────────┘
```

**组件说明**：
- 顶部导航：Logo + 当前用户 + 工单历史入口
- 消息列表：区分用户消息和助手消息，支持Markdown渲染
- 消息附件：展示引用的政策文档、工单单号等
- 输入框：支持多行输入，Enter发送，Shift+Enter换行

**视觉风格**：
- 主色调：企业蓝 (#1890ff)
- 背景色：浅灰 (#f5f5f5)
- 消息气泡：用户消息在右侧，助手消息在左侧
- 字体：系统默认无衬线字体

## 5. 非功能性需求

| 需求项 | 指标 |
|--------|------|
| 响应时间 | 对话响应 < 5秒，资源查询 < 3秒 |
| 并发用户 | 支持至少10个并发用户 |
| 数据持久化 | 所有工单、对话历史需持久化到MySQL |
| 兼容性 | Chrome、Firefox、Edge最新版本 |

## 7. 项目范围边界

### 包含
- 三条核心业务链路的完整实现
- 基础的对话界面
- 模拟数据与模拟审批逻辑
- 简历级别的代码质量与文档

### 不包含（避免过度开发）
- 真实的用户认证系统（用模拟用户ID）
- 真实的负责人审批流程（用模拟逻辑）
- 复杂的权限管理
- 移动端适配（仅需桌面端）
- 性能优化到生产级别
- 容器化部署（可选，时间充裕再加）
