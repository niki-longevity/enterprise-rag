# 企业内部员工助手智能体 - 需求文档

## 1. 项目概述

### 1.1 项目背景
为企业员工提供一站式智能助手，通过自然语言对话解决政策咨询等日常办公需求。

### 1.2 项目目标
- 实现核心链路：政策问答（RAG）
- 提供友好的对话式交互界面
- 展示LangGraph智能编排与RAG技术的结合应用
- 作为简历项目，体现完整的全栈开发能力

---

## 2. 用户角色

| 角色 | 描述 |
|------|------|
| 普通员工 | 使用助手查询政策 |

---

## 3. 功能需求

### 3.1 核心链路：IT政策问答（RAG）

**功能描述**：员工通过自然语言询问公司政策，系统基于RAG检索相关文档并回答。

**典型场景**：
- "婚假能请几天？"
- "我想换个新电脑，公司的相关规定是什么？"
- "请假流程是怎样的？"

**验收标准**：
- [x] 支持至少5个政策文档的检索
- [ ] 回答需引用对应的政策来源
- [ ] 当问题超出政策范围时，应礼貌告知
- [ ] 响应时间 < 5秒

### 4.2 聊天主页面（核心页面）

**布局结构**：
```
┌─────────────────────────────────────────────────┐
│  企业员工助手  🤖                          │
├─────────────────────────────────────────────────┤
│                                                 │
│  [历史消息区域]                                 │
│  ┌─────────────────────────────────────────┐   │
│  │ 员工：婚假能请几天？                  │   │
│  └─────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────┐   │
│  │ 助手：根据《休假管理规定》第3.2条...    │   │
│  │ 📄 来源：休假管理规定.md                │   │
│  └─────────────────────────────────────────┘   │
│                                                 │
│  [输入区域]                                     │
│  ┌─────────────────────────────────────────┐   │
│  │ 输入您的问题...                   [发送] │   │
│  └─────────────────────────────────────────┘   │
│                                                 │
└─────────────────────────────────────────────────┘
```

**组件说明**：
- 顶部导航：Logo + 当前用户
- 消息列表：区分用户消息和助手消息，支持Markdown渲染
- 消息附件：展示引用的政策文档
- 输入框：支持多行输入，Enter发送，Shift+Enter换行

**视觉风格**：
- 主色调：企业蓝 (#1890ff)
- 背景色：浅灰 (#f5f5f5)
- 消息气泡：用户消息在右侧，助手消息在左侧
- 字体：系统默认无衬线字体

## 5. 非功能性需求

| 需求项 | 指标 |
|--------|------|
| 响应时间 | 对话响应 < 5秒 |
| 并发用户 | 支持至少10个并发用户 |
| 数据持久化 | 对话历史需持久化到MySQL |
| 兼容性 | Chrome、Firefox、Edge最新版本 |

## 7. 项目范围边界

### 包含
- 政策问答链路的完整实现
- 基础的对话界面
- 简历级别的代码质量与文档

### 不包含（避免过度开发）
- 真实的用户认证系统（用模拟用户ID）
- 资源查询功能
- 审批流功能
- 复杂的权限管理
- 移动端适配（仅需桌面端）
- 性能优化到生产级别
- 容器化部署（可选，时间充裕再加）
