# 企业内部员工助手智能体 - 架构文档

> **状态**: 开发中

---

## 1. 技术选型

| 层级 | 技术选型 | 选型原因 |
|------|---------|---------|
| 后端服务 | Python + FastAPI | 异步高性能，Python AI生态好 |
| 智能编排 | LangGraph (ReAct模式) | 多步骤Agent编排，状态管理 |
| 大模型 | DeepSeek (OpenAI兼容接口) | 国内访问稳定，成本可控 |
| 向量库 | ChromaDB | 轻量本地向量库，开发简单 |
| Embedding | 阿里云DashScope | 国内服务稳定 |
| 数据库 | MySQL + SQLAlchemy ORM | 关系型存储对话历史，ORM简化CRUD |
| 前端 | React + TypeScript + Ant Design | 组件丰富，开发效率高 |

## 4. 核心模块说明

### 4.1 LangGraph ReAct流程

```
用户消息 → [Agent节点] ──需要工具──→ [Tool节点] → [Agent节点] → ... → 结束
                │                                              ↑
                └────────────不需要工具──→ 直接返回回答 ────────┘
```

- **Agent节点**：LLM（DeepSeek）判断是否需要调用工具，需要则返回tool_calls
- **Tool节点**：执行search_policy工具，检索ChromaDB向量库
- **should_continue路由**：检查LLM返回是否有tool_calls，有则走工具，无则结束

### 4.2 API接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/chat | 发送消息，返回AI回复 |
| GET | /api/history?session_id=xxx | 查询会话历史 |
| GET | /api/sessions?user_id=xxx | 查询用户的会话列表 |
| GET | /health | 健康检查 |

### 4.3 数据库交互

采用类似MyBatis Plus的BaseMapper封装模式：
- **BaseMapper**：通用CRUD（save、get_by_id、list_all、list_by_field）
- **ChatHistoryMapper**：继承BaseMapper，扩展业务查询（按会话查历史、按用户查会话列表）
- 对话历史在API层保存，调用agent前后各存一条（用户消息 + 助手回复）

## 5. 配置项

通过环境变量配置，`settings.py`中读取：

| 配置项 | 环境变量 | 默认值 | 说明 |
|--------|---------|--------|------|
| DASHSCOPE_API_KEY | DASHSCOPE_API_KEY | - | 阿里云DashScope API Key |
| DEEPSEEK_API_KEY | DEEPSEEK_API_KEY | - | DeepSeek API Key |
| DB_HOST | DB_HOST | localhost | MySQL地址 |
| DB_PORT | DB_PORT | 3307 | MySQL端口 |
| DB_USER | DB_USER | root | MySQL用户名 |
| DB_PASSWORD | DB_PASSWORD | 88888888 | MySQL密码 |
| DB_NAME | DB_NAME | db_ea | 数据库名 |
